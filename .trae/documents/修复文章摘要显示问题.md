## 问题分析

**现象**：文章摘要显示为HTML转义后的内容，如 `&lt;h1&gt;Python 编程基础入门&lt;/h1&gt;`，而不是渲染后的HTML。

**原因**：
1. 后端 `app.py` 的 `get_posts()` 函数中，summary 是通过 `markdown.markdown(content)` 生成的HTML内容
2. 前端 `Home.tsx` 中，摘要直接通过 `{post.summary}` 渲染，导致HTML标签被转义

## 解决方案

### 方案1：修改前端使用 dangerouslySetInnerHTML（不推荐）
- **优点**：修改简单，只需要修改前端
- **缺点**：存在XSS安全风险，不符合React最佳实践

### 方案2：修改后端生成纯文本摘要（推荐）
- **优点**：安全可靠，符合最佳实践
- **缺点**：需要修改后端逻辑

## 实施计划

1. **修改后端 `app.py`**：
   - 修改 `get_posts()` 函数，在生成摘要时移除HTML标签
   - 使用正则表达式或文本处理库提取纯文本

2. **测试修复效果**：
   - 重启Flask服务
   - 访问前端页面，确认文章摘要显示正常

3. **验证其他功能**：
   - 确保API其他端点正常工作
   - 确保文章列表和详情页显示正常

## 技术细节

在 `app.py` 中，将：
```python
# 生成文章摘要
html_content = markdown.markdown(content)
summary = html_content[:200] + '...' if len(html_content) > 200 else html_content
```

修改为：
```python
# 生成文章摘要
html_content = markdown.markdown(content)
# 移除HTML标签，生成纯文本摘要
import re
plain_text = re.sub('<[^<]+?>', '', html_content)
summary = plain_text[:200] + '...' if len(plain_text) > 200 else plain_text
```
